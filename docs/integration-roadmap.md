### リスク予算(キーストーン・確定)

```
固定の床(ルイン・ライン)= 未来予測に依存しない死守線
  生活防衛 400 + 教育予約 200(仮置き・#2未確定) + 複利装置(NISA+旧) 381 ≈ 980万
  → 総資産が約980万を割る事態は何があっても作らない

張っていい部屋(上限)= 1,514 − 980 ≈ 530万
  → 天井であってノルマではない。実際に張る量は「証明されたエッジの分だけ」

現在リスクに張っている額 = crypto 200 + 特定 15 ≈ 215万(部屋の半分以下)
```

ユーザー確認済みの覚悟: 「crypto 200万が一晩でゼロでも計画は無傷、織り込み済み」。
ただし「400万は驚く」=エッジ未証明の今、部屋上限まで張るのは早い(=正しい)。

### 致命的発見: 全サイロ、実現エビデンスがゼロ

| サイロ | 実現往復 | 税後レイヤー | 自己評価の足場 |
|---|---|---|---|
| crypto アルト | 0(4 BUY 全部 OPEN・一度も売ってない) | なし | profit_loss 列はあるが空 |
| crypto BTC | 0(稼働10日・5/20〜) | なし | 同上 |
| 株 realized_trades | 0 | account_type で税率分岐可 | realized_gain あり |
| 株 exit_judgments | 28(=出口判定の正誤評価) | なし(pct は素値) | decision_was_right/pct_at_horizon あり |

→ **今この瞬間、crypto も株アクティブもデータ上は "賭け"**(実現エビデンスなし)。
→ よって操縦席が今やるのは「再配分」ではなく「証拠が溜まる器を作ること」。

### Phase 1 — 計測コントラクトの定義 ✅ 完了(2026-05-30)
→ 完成物: [integration-measurement-contract.md](./integration-measurement-contract.md)
**重要な修正(crypto売りロジック調査の結果):** 全サイロを「同一の実現往復EV」で測るのは破綻する。
crypto は暴落買い戦略で **5年で約14-15約定**しかせず、実現EVが溜まるのに年単位かかる。
→ **共通の出力単位は1つに揃え、推計方法はサイロごとに変える + 証拠グレードのタグを付ける。**

- [x] 共通の出力単位: **資本あたり・年率・税後の期待リターン**(これで横並び可能に)
- [x] サイロ別の推計方法:
  - 株アクティブ(特定・20日回転): 実現往復の税後EV(realized_trades)+ exit_judgments を先行指標。数ヶ月で溜まる
  - crypto(暴落買い・年数回): 実現は待てない → ①バックテストCAGR30-35% ②**ライブCAGR vs BT追従**(`monthly_snapshots.cagr_running`)③含み時価健全性
  - NISAコア(オルカン買い持ち): 取引でない → 期待市場リターン ~5-7%
- [x] 税率テーブル: NISA 0% / 特定 20%(損益通算考慮) / crypto 実効20〜33%(山田家実所得ベース・integration-measurement-contract.md 参照)
- [x] 証拠グレード: 実証(realized N十分) > バックテスト > 理論値。**操縦席は低グレードに寄せない**
- [x] 出力先スキーマ設計(統合 view。crypto は自前 `monthly_snapshots` を翻訳するだけ=配管軽い)

### ★ North Star(2026-05-30 ユーザー指摘): 出口判断の質が系全体のレバレッジ点
「買い判断は緩くてよい。利確/損切り/長期保有判断(=出口)を確率計算・損益計算で磨けばリターン最大化」。
出口判断の質を1上げると crypto も株も τ が同時に上がる(両サイロが出口を通じてτを出すため)。
→ **cross-silo 出口品質スコアカード**が最優先の計器。測れないものは改善できない。
